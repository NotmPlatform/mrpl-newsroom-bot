# MRPL Newsroom Bot

Telegram-редакция для MRPL.RU:

- администратор назначает редакторов и волонтёров по Telegram ID;
- автор присылает текст или голосовое сообщение и до 10 фотографий;
- Yandex SpeechKit расшифровывает голос;
- DeepSeek создаёт фактическую редакторскую версию и SEO-поля;
- волонтёр подтверждает материал до передачи редактору;
- редактор получает ссылку на черновик WordPress;
- публикация выполняется одной кнопкой и не затирает ручные правки WordPress;
- отклонение требует комментария автору;
- редактор может самостоятельно создать и сразу опубликовать материал;
- все действия сохраняются в PostgreSQL.

## 1. WordPress

1. Установите и активируйте `mrpl-news-connector.zip`.
2. Создайте пользователя `mrpl_news_bot`.
3. Назначьте роль **MRPL News Bot**.
4. В профиле пользователя создайте пароль приложения `MRPL Railway Bot`.
5. Не назначайте этому пользователю роль администратора.

## 2. Telegram

1. Создайте бота через `@BotFather`.
2. Сохраните токен.
3. Команды бот установит автоматически.

## 3. Yandex SpeechKit

1. Создайте сервисный аккаунт.
2. Назначьте роль `ai.speechkit-stt.user`.
3. Создайте API-ключ со scope `yc.ai.speechkitStt.execute`.
4. Сохраните ID каталога и секрет API-ключа.

Голосовые сообщения до 5 минут автоматически делятся `ffmpeg` на фрагменты до 28 секунд.

## 4. DeepSeek

Создайте API-ключ в кабинете DeepSeek. По умолчанию используется
`deepseek-v4-flash`.

## 5. GitHub и Railway

1. Создайте приватный GitHub-репозиторий.
2. Загрузите в корень содержимое этого проекта.
3. В Railway создайте проект.
4. Добавьте PostgreSQL.
5. Создайте сервис из GitHub-репозитория.
6. Установите ровно одну реплику.
7. Публичный домен и Railway Volume не нужны.
8. В Variables добавьте значения из `.env.example`.
9. Для `DATABASE_URL` используйте Reference на переменную PostgreSQL.
10. Pre-deploy command: `alembic upgrade head`.
11. Start command: `python -m app.main`.

## 6. Первый запуск

1. Сначала оставьте `ADMIN_TELEGRAM_ID=0` и `PUBLISH_ENABLED=false`.
2. Откройте бота, нажмите Start и выполните `/myid`.
3. Запишите полученный ID в `ADMIN_TELEGRAM_ID`.
4. Выполните Redeploy.
5. Проверьте `/status`.

## 7. Роли

```text
/role 123456789 editor
/role 987654321 volunteer
/team
/revoke 987654321
```

Каждый пользователь должен сам открыть бота и нажать Start, иначе Telegram
не позволит отправлять ему уведомления.

## 8. Проверка перед включением публикации

1. Отправьте тестовую новость как волонтёр.
2. Проверьте редакторскую версию.
3. Подтвердите передачу.
4. Откройте созданный черновик WordPress.
5. Проверьте заголовок, текст, обложку, галерею, рубрику и SEO-поля.
6. Проверьте отклонение с комментарием.
7. Проверьте повторное нажатие кнопок.
8. Только после полного теста установите `PUBLISH_ENABLED=true`.

## 9. Резервные копии

В Railway включите ежедневные и еженедельные backups PostgreSQL. Перед
обновлением кода создавайте ручной backup.

## Безопасность

- репозиторий только приватный;
- `.env` никогда не коммитить;
- секреты хранить только в Railway Variables;
- WordPress и API вызываются только по HTTPS;
- при утечке отозвать соответствующий API-ключ или Application Password;
- фотографии пересохраняются в JPEG без EXIF/GPS;
- DeepSeek не получает фотографии, токены Telegram или пароли;
- публикация выключена до явного `PUBLISH_ENABLED=true`.
